# 推荐文章功能临时测试方案（仅主页，可删除）

## 1. 目标
验证以下功能是否正确：
- 仅在「主页」展示推荐文章（标题 + 图片）
- 「MBTI」页面不展示推荐文章栏
- 在主页点击文章可进入详情页
- 详情页可看到标题、图片、正文

## 2. 测试前提
- 使用微信开发者工具打开项目
- 编译成功，无红色报错
- 当前示例数据在 `miniprogram/data/article-repository.js`

## 3. 测试用例（手工）

### 用例 A：主页文章列表展示
1. 打开 Tab「主页」。
2. 观察推荐文章栏。
3. 预期结果：
   - 至少看到 1 条文章卡片
   - 每条卡片显示标题“标题”
   - 每条卡片有封面图

### 用例 B：MBTI 页面不显示文章栏
1. 打开 Tab「MBTI」。
2. 观察页面内容。
3. 预期结果：
   - 页面不出现推荐文章卡片列表
   - 页面仅显示 MBTI 页面内容占位信息

### 用例 C：主页点击进入详情
1. 在「主页」点击任意文章卡片。
2. 预期结果：
   - 跳转到文章详情页
   - 页面显示标题“标题”
   - 页面显示正文“正文”
   - 页面显示封面图

### 用例 D：异常 id 兜底（可选）
1. 在开发者工具控制台执行：
   `wx.navigateTo({ url: '/page/article/index?id=not-exists' })`
2. 预期结果：
   - 出现提示：`Article not found`
   - 页面不崩溃

## 4. 通过标准
- 用例 A/B/C 全部通过即功能可验收
- 用例 D 通过表示异常处理可用（建议通过）
